# Global Asset Library

Assets are SVG files shared across channels and scenarios. The library lives at `pipeline/global_assets/` and is indexed in `pipeline/global_assets/assets_index.json`.

## Asset ID Format

Every asset gets an auto-generated ID when uploaded:

```
G-CHR-001-001-BODY
‚îÇ  ‚îÇ   ‚îÇ   ‚îÇ   ‚îî‚îÄ‚îÄ Role suffix
‚îÇ  ‚îÇ   ‚îÇ   ‚îî‚îÄ‚îÄ Asset number within group
‚îÇ  ‚îÇ   ‚îî‚îÄ‚îÄ Group number within type
‚îÇ  ‚îî‚îÄ‚îÄ Type code
‚îî‚îÄ‚îÄ Scope prefix
```

**Scope prefix:**
- `G-` ‚Äî global asset (in `pipeline/global_assets/`)
- `P-` ‚Äî project-specific asset (in the scenario workspace)

**Type codes:**
- `CHR` ‚Äî character
- `OBJ` ‚Äî object or prop
- `BG` ‚Äî background
- `BUB` ‚Äî speech bubble
- `SND` ‚Äî sound effect
- `MUS` ‚Äî music
- `SFX` ‚Äî sound FX

**Role suffixes:**
- `BASE` ‚Äî canonical reference (one per group ‚Äî enforced by the server)
- `BODY` ‚Äî character body part
- `FACE` ‚Äî face/head
- `EYES` ‚Äî eye state
- `CTX` ‚Äî context element (default for most objects)
- `PART` ‚Äî partial component
- `PROP` ‚Äî physical prop
- `LORA` ‚Äî LoRA training target

## Asset Tags

Each asset entry in `assets_index.json` carries:

| Field | Type | Description |
|---|---|---|
| `semantic_tags` | string[] | Concept tags: `["money", "wealth", "stress"]` |
| `emotion_tags` | string[] | Emotional resonance: `["tension", "urgency", "fear"]` |
| `visual_energy` | float 0‚Äì1 | How visually active this asset is (0 = calm, 1 = high energy) |
| `attention_weight` | float 0‚Äì1 | How much viewer attention this asset draws |
| `global_uses` | int | Total uses across all projects |
| `project_uses` | int | Uses in the current project |
| `compatible_groups` | string[] | Group IDs this asset can be combined with |

**Overuse thresholds** (configurable in `engines.yaml`):
- `global_warn_threshold`: 10 ‚Äî warns if an asset has been used across 10+ videos
- `project_warn_threshold`: 5 ‚Äî warns if an asset is reused 5+ times in one video
- `exempt_roles`: `["BASE", "LORA"]` ‚Äî these roles bypass overuse checks

## How to Add an Asset

### Via tool (Claude-driven)

```
pipeline_upload_asset(
    channel=..., scenario=...,
    category="characters/crowd/body",
    name="standing_confident",
    svg_content="<svg ...>#PLACEHOLDER_COLOR...</svg>",
    scope="global",
    role="BODY",
    semantic_tags='["confidence", "authority"]',
    emotion_tags='["calm", "secure"]',
    visual_energy="0.3",
    attention_weight="0.6"
)
```

The server auto-generates the ID and registers the asset in `assets_index.json`.

### SVG conventions

- Use `#PLACEHOLDER_COLOR` (not a hex value) for character skin/fill color. The compositor replaces this with the channel's color palette at render time.
- Target viewport: 1920√ó1080 for backgrounds; variable for characters and objects.
- Keep paths clean ‚Äî CairoSVG renders the SVG; complex filters may not render correctly.

## SVG Character Parts (Stickman Rig)

Characters are assembled from modular SVG parts at render time by `Stickman.tsx`:

```
characters/crowd/
‚îú‚îÄ‚îÄ body/        One file per pose: standing, sitting, walking, pointing, etc.
‚îú‚îÄ‚îÄ eyes/        One file per state: open, wide, closed, squint
‚îî‚îÄ‚îÄ mouth/       One file per phoneme: A, B, C, D, E, F, G, H, X (silence)
```

**Color override:** All parts use `currentColor` or `#PLACEHOLDER_COLOR`. At render time, Remotion passes `color=` from `channel_config.json ‚Üí visual_style.character_color`. This lets a single SVG set serve multiple channels by changing only the config.

**Assembling a character in a scene event:**

```json
{
  "time": 0.5,
  "action": "show",
  "target": "worker",
  "state": {
    "emotion": "neutral",
    "pose": "standing",
    "asset_group": "G-CHR-001"
  }
}
```

`Stickman.tsx` resolves `asset_group` to the correct body/eyes/mouth SVG files, then applies lipsync mouth-shape overrides if `lipsync_file` is set on the scene.

## Searching Assets

```
pipeline_search_assets(
    channel=..., scenario=...,
    semantic="money wealth",
    emotion="tension",
    scope="global",
    role="CTX"
)
```

Returns assets matching ALL provided filters. Combine `semantic`, `emotion`, `role`, and `asset_type` filters to narrow results. Results include `global_uses` and `project_uses` so Claude can avoid overuse.
